CRITICAL IMPLEMENTATION DETAILS
════════════════════════════════════════════════════════════════════════════════

1. TIMING DELTA CALCULATION (Most Important):
   delta = input_time - note_time
   
   - Positive = hit too late (slow)
   - Negative = hit too early (fast)
   - Zero = perfect timing

2. HIT WINDOW VALIDATION (Second Most Important):
   if abs(delta) <= 0.040:  # Perfect window
       return PERFECT
   elif abs(delta) <= 0.080:  # Great window
       return GREAT
   # ... etc

3. MATCHED NOTE TRACKING:
   MUST mark notes as matched immediately to prevent double-matching
   This is critical for simultaneous input handling

4. MISSED NOTE CLEANUP:
   Run every frame: if current_time - note_time > LATE_WINDOW: mark_as_missed()
   This prevents audio gaps where notes weren't hit

════════════════════════════════════════════════════════════════════════════════
COMMON MISTAKES TO AVOID
════════════════════════════════════════════════════════════════════════════════

❌ Mistake 1: Checking all notes instead of searching nearby
   Impact: O(n) complexity becomes slow on larger songs
   Fix: Use search window (±0.2s) to only check candidate notes

❌ Mistake 2: Not marking matched notes
   Impact: Same note can be hit multiple times
   Fix: Set note.matched = true immediately after hitting

❌ Mistake 3: Calculating delta as abs(input - note) only
   Impact: Can't tell if hit was early or late
   Fix: Use signed delta: input_time - note_time

❌ Mistake 4: No latency compensation
   Impact: Hits feel "off" or unfair
   Fix: Subtract audio latency offset from input time

❌ Mistake 5: Tight hit windows without calibration
   Impact: Players can't consistently hit notes
   Fix: Make windows calibratable by player

❌ Mistake 6: Checking all notes every frame for missed
   Impact: Wastes CPU cycles checking old notes repeatedly
   Fix: Only check notes within reasonable range
